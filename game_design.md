# Game Design Document (OLED)

## 1. ゲーム概要
- ジャンル: 探索・タイムアタック
- 想定プレイ時間: 1プレイ 1〜3分程度（全ステージ）
- コア体験（何が楽しいか）:
  - 複数の `shimaenaga` の中から `pochacco` を素早く見つける。
  - 面クリアを重ねて、全ステージの合計クリア時間を縮める。
- 1行コンセプト: 「似たキャラクター群から `pochacco` を見つける、OLED向け高速探索ゲーム」
- ゲームの流れ:
  1. 1つの `pochacco.c` と複数の `shimaenaga.c` を画面内に表示する。
  2. プレイヤーは `up` / `down` / `left` / `right` ボタンでカーソルを移動する。
  3. カーソル位置と `pochacco` の位置が一致したらその面をクリアする。
  4. 面クリア時間を記録し、次の面へ遷移する。
  5. 全ての面をクリアした合計時間を最終スコアとして競う。

## 2. 操作仕様
- 入力デバイス（ボタン/ジョイスティック等）: `up`, `down`, `left`, `right` の4ボタン
- 画面グリッド: `? x ?` マスのタイル（ステージごとにサイズ可変）
- 初期カーソル位置: `2 x 1`
- ボタン割り当て:
  - `up`: カーソルを上へ1マス移動
  - `down`: カーソルを下へ1マス移動
  - `left`: カーソルを左へ1マス移動
  - `right`: カーソルを右へ1マス移動
- 長押し/連打の扱い: 未定義（実装時に調整）
- 同時押し対応: なし（単一入力を優先）
- ステージ遷移条件: カーソル座標と `pochacco` の座標が一致したら次ステージへ移動

## 3. ゲームルール
- ステージ構成: `1 ～ N` ステージ（`N` は後で確定）
- 目的（勝利条件）: 各ステージで `pochacco` を見つけて、全ステージをクリアする
- 敗北条件: なし（タイムアタック形式）
- スコア計算: 全ステージクリアまでの合計時間をスコアとする（短いほど良い）
- 全ステージ共通仕様:
  - `pochacco` は1つだけ配置する
  - 残りタイルは `shimaenaga` を配置する
  - 開始位置 `2 x 1` には `pochacco` を配置しない
  - `pochacco` の位置はステージ開始ごとにランダムで決定する
- ステージ1仕様:
  - `5 x 3` タイル（チュートリアル）
  - ディスプレイサイズに合わせて、1タイルを大きめに表示する
- ステージ2仕様:
  - `7 x 5` タイル
  - キャラクターの向きはデフォルト
- ステージ3仕様:
  - `7 x 5` タイル
  - キャラクターの向きは `90` / `180` / `270` 度のいずれかに変更する
  - 全てのタイルを同じ向きにそろえる
- ステージ4仕様:
  - `7 x 5` タイル
  - キャラクターの向きはデフォルト
  - 白黒を反転して表示する
- ステージ5仕様:
  - `7 x 5` タイル
  - キャラクターの向きはタイルごとにランダムで変更する
- ステージ6仕様:
  - `7 x 5` タイル
  - キャラクターの向きはデフォルト
  - プレイヤーが操作するたびに、`pochacco` の位置をランダムで移動する
- ステージ7仕様:
  - `7 x 5` タイル
  - キャラクターの向きはデフォルト
  - カーソルを中心に `3 x 3` 相当（円形）の範囲のみ表示する
  - 表示範囲外はマスクして非表示にする

## 4. ゲーム状態（ステート）
- `STATE_TITLE`:
  - タイトルを表示し、ゲーム開始待ちの状態
- `STATE_STAGE_START`:
  - ステージ番号（例: `STAGE 1`）を短時間表示して開始演出を行う状態
- `STATE_PLAY`:
  - プレイヤー操作を受け付け、現在ステージの経過タイムを表示する状態
- `STATE_STAGE_CLEAR`:
  - ステージクリアを表示し、次ステージへの遷移判定を行う状態
- `STATE_ALL_CLEAR`:
  - 全ステージクリア後、合計タイムを表示する最終結果状態

状態遷移:
- `STATE_TITLE` -> `STATE_STAGE_START`
- `STATE_STAGE_START` -> `STATE_PLAY`
- `STATE_PLAY` -> `STATE_STAGE_CLEAR`（`pochacco` 発見時）
- `STATE_STAGE_CLEAR` -> `STATE_STAGE_START`（次ステージがある場合）
- `STATE_STAGE_CLEAR` -> `STATE_ALL_CLEAR`（最終ステージクリア時）
- `STATE_ALL_CLEAR` -> `STATE_TITLE`（任意キーでタイトルへ戻る）

- OLED表示ルール:
  - ステージごとのグリッドサイズ（例: `3x3`, `5x5`）に応じてタイルを自動レイアウトする
